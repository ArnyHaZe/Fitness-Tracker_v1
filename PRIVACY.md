# Privacy Policy

Fitness Tracker v1 is designed as a local-first app.

## Data Collection

This app does not collect personal data.

There are:

- No user accounts
- No login system
- No analytics scripts
- No tracking pixels
- No backend server
- No cloud database

## Where Data Is Stored

Workout logs, check-ins, exercise swaps, phase selections, and progress data are stored locally in the browser using localStorage.

This means the data remains on the device and browser where the app is used.

## Data Sharing

No data is transmitted to the developer or to any external database.

The app loads Chart.js from jsDelivr CDN. That CDN may receive standard web request information such as IP address and browser user agent when the library is loaded.

## Deleting Data

Users can delete saved progress by clearing site data for the app in their browser settings.

Important: clearing browser data can permanently remove saved workout progress unless an export or backup feature is added in the future.

## Offline Use

The service worker caches app files on the device so the app can open offline after the first visit.

## Contact

For questions or issues, open a GitHub issue in this repository.
